# hiroto-fire-blog 運用ルール

## 文体・温度感ルール

### 基本方針
「内容はしっかり・でもクスッと笑えてツッコミたくなる」文体で書く。
ふざけすぎず、真面目すぎず。読んでいて疲れない温度感を優先する。

### 改行ルール
- スマホで読まれる前提で、改行は多めに入れる
- 1段落は原則1〜3文まで。4文以上つなげない
- 大事な一文、感情が動く一文、話題が切り替わる一文の後は改行する
- ブログ本文もスマホで読みやすい改行を優先し、1段落を重くしすぎない

## 刺さる読者方針

### 想定読者
- 40代会社員
- 副業初心者
- お金に強くない
- 面倒なことを後回しにしがち
- 将来や子どものために不安がある
- でも何か変えたいと思っている人

### 書き方の基準
- 広く無難に届く記事より、ひろとに近い状況の読者に深く刺さる記事を優先する
- 一般論を薄く広く書かず、「うわ、それ自分のことだ」と思う具体的な失敗、迷い、後回し、焦りを書く
- 体験談でも日記で終わらせず、読者が持ち帰れる学びを入れる

## 案件記事の文体ルール

### 毎回入れる要素
1. ベネフィットは具体的に書く
   - 「便利です」で終わらせず、時間・手間・精神的な負担の変化まで書く
2. ズボラ感を1箇所は入れる
   - 面倒で逃げた、後回しにした、雑だった、を言葉を変えながら入れる
3. ポンコツな僕でもできた感を入れる
   - 知識なし、経験なし、挫折済みでも進められた流れを書く
4. クスッとする自己ツッコミを自然に入れる
   - カッコに逃げず、文の流れの中で自然に入れる

### NG
- 「非常に重要なポイントです」「ぜひ活用してみてください」などの煽り
- テンションが高すぎる文体
- 説教・上から目線
- 笑いが多すぎて内容が薄く見える記事
- ボケのための作り話

## 記事設計方針

### 記事執筆ルール（完全版）

- 1記事1悩みに絞る。広く書くと薄くなる
- 正しさだけでなく、人間味が残る記事を優先する
- 一般論より「その人の夜に刺さる具体」を書く
- 最後は要約だけで終わらせず、次の1アクションまで軽く示す

### 記事タイプの使い分け
- 体験談記事：共感・信頼・人間味を作る
- 集客記事：検索流入を取ってまとめ記事やキラーページへ流す
- まとめ記事：全体像を伝え、詳細記事へ回遊させる
- レビュー記事：実体験・使用感・失敗談を入れて訴求する
- 比較記事：違い・向いている人・選び方を明確にする
- ランキング記事：読者の代わりに調べて順位づけし、選ぶ手間を減らす

### 案件記事を増やしすぎないルール
- 案件ごとにレビュー記事を量産しない
- 大事なのは案件記事を増やすことではなく、案件にたどり着くまでの悩み記事を増やすこと
- 記事タイプの割合は「悩み解決・集客記事50% / 体験談・信頼記事30% / レビュー・比較・成約寄り記事20%」を基準にする
- 1案件につき、必ずレビュー1本だけを書くのではなく、読者の悩みを起点に周辺記事を作る
- すべての案件を同じ密度で展開しない

### 構成ルール
- リード文は原則「悩み → 解決策/結論 → ベネフィット」
- リード文は本文を書き切ったあとに調整する
- H2は読者の疑問に答える順番に並べる
- 記事末は「まとめ + 次の行動」で終える
- タイトルと同じ文言を、本文冒頭の見出しで繰り返さない

### 書き出しルール
- 「副業が大事です」のような一般論ではなく、読者が自分ごと化しやすい情景から入る
- うまくいった話だけでなく、閉じた、後回しにした、面倒で止まった、少し恥ずかしかった、を残す
- 導入で正しさを急ぎすぎず、先に人間味を出す

### 執筆前チェック
- 記事タイプを先に決める
  - 悩み解決
  - 体験談
  - レビュー
  - ハブ
- SEO記事はキーワードボリュームを確認してから着手する
- 甘い話を盛るより、現実的な評価と判断材料を優先する
- ひろとの実体験と実際の言葉を削りすぎない

### 面白さ・温度感の実務ルール
- 正しい説明だけで終わらせず、各記事に「その人の夜が見える一文」を入れる
- 導入、各H2の最初の2〜3文、まとめ前後は特に温度感を強める
- 失敗、後回し、恥ずかしさ、小さい自己ツッコミを自然に混ぜる
- カッコ内オチは禁止。文の流れの中でクスッとさせる
- 「覚えて残る記事」を優先し、情報7：人間味3を目安にする

### 案件・導線ルール
- 結論は先に、はっきり伝える
- ベネフィットは抽象語で終わらせず、数字・手間・時間・気持ちの変化まで書く
- 読者が「自分はどうするか」を判断できる具体的な基準を入れる
- 記事末は売り込みすぎず、次の1アクションを軽くする

### 記事内画像ルール（2026-05-31更新・必須化）
- **すべての記事に本文内SVG画像を最低1枚入れる（必須）**
- 画像なしの記事は採点で画像・サムネ軸が3点以下になるため入稿基準を満たしにくい
- 優先する画像タイプ
  - Before/After比較図（変化・効果を見せる）
  - 手順の流れ図（ステップを整理する）
  - 比較表の図解（違いを一目で見せる）
  - 判断基準の整理図（「自分はどっち？」を助ける）
- SVGは `/public/images/articles/[slug]/[画像名].svg` に保存する
- 記事内での挿入は `![alt](/images/articles/[slug]/[画像名].svg)` 形式
- 挿入箇所：記事の中盤（体験談や手順の後）が最も効果的
- 画像は飾りではなく「読む負担を減らす・理解を助ける」目的で入れる

## FAQの入れる・入れないルール

### 基本方針
FAQは全記事に入れない。
記事タイプによって判断する。

### 入れる記事タイプ
- 確定申告・税金・手順系：条件や数字が複雑で具体的な疑問が出やすい
- サービスレビュー系：料金・機能・使い方の疑問が出やすい
- 比較・まとめ系：疑問が出やすい場合のみ

### 入れない記事タイプ
- 体験談・感情系：温度感が冷める・本文で完結している
- 考え方・マインド系：同上
- 日常・シングル父系：同上

### 判断基準
「本文を読んだあとに、まだ具体的な疑問が残るか？」
残るなら入れる。本文で完結しているなら入れない。

## SEO・タイトルチェック基準

### SEO記事（検索流入狙い）に適用
1. タイトルに検索キーワードを入れる（読者が実際に検索する言葉）
2. 同じキーワードを狙った記事が複数ないか確認する（カニバリゼーション防止）
3. タイトルは27文字以内にする（スマホ表示で切れないため）
4. descriptionは80〜120文字程度にする
5. 抽象的なタイトルにしない（誰も検索しない言葉はNG）

### 感情系・体験談記事には適用しない
- 感情系・体験談記事はSNS流入が主なので「刺さるタイトル」優先
- SEO基準より共感・人間味を重視する

### 新規記事着手前に必ず確認
- ラッコキーワード・Googleトレンドでキーワードボリュームを確認する
- 月間100以下または競合が強すぎる場合はひろとさんに確認を取る

## 記事作成の鉄則

### 1記事1キーワード
- 1つの記事で狙うキーワードは1つだけ
- 似たキーワードは1つの記事にまとめる
- 同じキーワードで複数記事を作らない（カニバリゼーション厳禁）

### 低品質記事を作らない
- 似た内容の記事を量産しない
- 体験談・数字・具体的な手順のない記事は作らない
- 記事を作る前に既存記事と内容が被らないか確認する

### リライトの基準
- 古い情報は最新情報に変更する
- タイトルと見出しを定期的に見直す
- 読者の問題と解決策を冒頭に書く
- 体験談などの独自情報を追加する
- 上位ページと比較してコンテンツを充実させる
- リライトした記事に内部リンクを張る

### タイトルの書き方
- 27文字以内に収める
- 数字を入れる
- 読むメリットを入れる
- 抽象的な言葉を避ける
- 1ページにつき1キーワードを前半に入れる

## 表記ルール

### 「シングル父」→「シングルファーザー」
- 新規記事から「シングルファーザー」に統一する
- 既存記事はリライト時に順次変更
- 感情系・体験談記事は「シングル父」のままでもOK
- タイトル・description・見出しは必ず「シングルファーザー」を使う

- 40代会社員

- お金に強くない

### ADSENSE_REVIEWタグの運用ルール（2026-05-31追加）
- アドセンス審査中は `<!-- ADSENSE_REVIEW_START -->` と `<!-- ADSENSE_REVIEW_END -->` の間は**空にする**
- リンクや導線テキストは一切書かない（書いても表示されてしまうため）
- 審査通過後は「戻して」の一言で一括復元する
- 審査通過後に入れる内容（アフィリリンク・LINEボタン・LINE導線）はコメントとして別ファイルに保持しておく必要はない。リンクURLは既存記事から参照すればよい

### pushとデプロイのセット作業（2026-05-31追加）
- 記事を入稿するときは以下をセットで行う
  1. 記事ファイル作成
  2. サムネイル生成（`python3 scripts/generate_thumbnail.py --title "..." --category "..."`）
  3. git add・commit・push
  4. Cloudflareのデプロイ完了を確認
- サムネを忘れると記事にOGP画像が表示されない
