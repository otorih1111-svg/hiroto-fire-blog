# hiroto-fire-blog 運用ルール

## 文体・温度感ルール

### 基本方針
「内容はしっかり・でもクスッと笑えてツッコミたくなる」文体で書く。
ふざけすぎず、真面目すぎず。読んでいて疲れない温度感を優先する。

### 改行ルール
- スマホで読まれる前提で、改行は多めに入れる
- 1段落は原則1〜3文まで。4文以上つなげない
- 大事な一文、感情が動く一文、話題が切り替わる一文の後は改行する
- ブログ本文もスマホで読みやすい改行を優先し、1段落を重くしすぎない

## 刺さる読者方針

### 想定読者
- 40代会社員
- 副業初心者
- お金に強くない
- 面倒なことを後回しにしがち
- 将来や子どものために不安がある
- でも何か変えたいと思っている人

### 書き方の基準
- 広く無難に届く記事より、ひろとに近い状況の読者に深く刺さる記事を優先する
- 一般論を薄く広く書かず、「うわ、それ自分のことだ」と思う具体的な失敗、迷い、後回し、焦りを書く
- 体験談でも日記で終わらせず、読者が持ち帰れる学びを入れる

## 案件記事の文体ルール

### 毎回入れる要素
1. ベネフィットは具体的に書く
   - 「便利です」で終わらせず、時間・手間・精神的な負担の変化まで書く
2. ズボラ感を1箇所は入れる
   - 面倒で逃げた、後回しにした、雑だった、を言葉を変えながら入れる
3. ポンコツな僕でもできた感を入れる
   - 知識なし、経験なし、挫折済みでも進められた流れを書く
4. クスッとする自己ツッコミを自然に入れる
   - カッコに逃げず、文の流れの中で自然に入れる

### NG
- 「非常に重要なポイントです」「ぜひ活用してみてください」などの煽り
- テンションが高すぎる文体
- 説教・上から目線
- 笑いが多すぎて内容が薄く見える記事
- ボケのための作り話

## 記事設計方針

### 記事タイプの使い分け
- 体験談記事：共感・信頼・人間味を作る
- 集客記事：検索流入を取ってまとめ記事やキラーページへ流す
- まとめ記事：全体像を伝え、詳細記事へ回遊させる
- レビュー記事：実体験・使用感・失敗談を入れて訴求する
- 比較記事：違い・向いている人・選び方を明確にする
- ランキング記事：読者の代わりに調べて順位づけし、選ぶ手間を減らす

### 案件記事を増やしすぎないルール
- 案件ごとにレビュー記事を量産しない
- 大事なのは案件記事を増やすことではなく、案件にたどり着くまでの悩み記事を増やすこと
- 記事タイプの割合は「悩み解決・集客記事50% / 体験談・信頼記事30% / レビュー・比較・成約寄り記事20%」を基準にする
- 1案件につき、必ずレビュー1本だけを書くのではなく、読者の悩みを起点に周辺記事を作る
- すべての案件を同じ密度で展開しない

### 構成ルール
- リード文は原則「悩み → 解決策/結論 → ベネフィット」
- H2は読者の疑問に答える順番に並べる
- 記事末は「まとめ + 次の行動」で終える
- タイトルと同じ文言を、本文冒頭の見出しで繰り返さない

## FAQの入れる・入れないルール

### 基本方針
FAQは全記事に入れない。
記事タイプによって判断する。

### 入れる記事タイプ
- 確定申告・税金・手順系：条件や数字が複雑で具体的な疑問が出やすい
- サービスレビュー系：料金・機能・使い方の疑問が出やすい
- 比較・まとめ系：疑問が出やすい場合のみ

### 入れない記事タイプ
- 体験談・感情系：温度感が冷める・本文で完結している
- 考え方・マインド系：同上
- 日常・シングル父系：同上

### 判断基準
「本文を読んだあとに、まだ具体的な疑問が残るか？」
残るなら入れる。本文で完結しているなら入れない。
